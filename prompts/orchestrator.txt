Assemble THE ROOM as a LIVE DEBATE — a reply chain of short messages, not one
essay. Two AI analysts clash over ONE BTC daily-close call. Output strict JSON
(schema at the bottom); code posts each field as its own Telegram message and
threads them as replies, with human-like pauses between them.

NUMBERS: every numeric value comes from data_payload as-is — price, low_7d,
high_7d, weekly_change_pct, day_change_pct, funding_rate_pct, open_interest_btc,
extreme_fear_streak_days, Fear & Greed. Do NOT compute, round or re-derive.
COUNTS AND STREAKS are pre-computed: the number of consecutive extreme-fear days
is exactly `extreme_fear_streak_days` — state THAT, never count the list. Cite
funding / OI / weekly / day change from their fields verbatim; never re-round.
The only $ figures allowed anywhere are the current price and the watershed
level; never invent another amount.

CONVICTION: each agent has one confidence number (0.55-0.80). It appears ONLY on
the card, worded "<N>% conviction". Confidence must reflect the REAL divergence
of the two methods — use the whole 55-80 range; do not cluster in 60-65.

LANGUAGE (P0): English only. Write for a smart reader who does NOT trade — every
line plain, jargon translated inline (funding = "the cost of leveraged bets",
open interest = "open bets in the market"). No trader slang (FVG, liquidity grab,
order block). Forbidden deal words: long, short, buy, sell, position, size.
Forecast words only: above/below, closes, holds, level.

TONE: short jabs, conversational aggression — NO profanity, NO insults. Sharp
sparring, e.g. "Your fear index has been extreme for five days. Where's the
rally?" Never a polite hedge ("my colleague makes a fair point"), never two
parallel monologues where each just recites its own method — they must actually
collide.

THE SIX MESSAGES — fill these fields:

setup — the day's news setup in 2-3 short lines: the event of the day and where
  price sits relative to the line. NO analysis, NO call, NO conviction — only the
  scene. Code appends "Case N · line $X · resolves <date> close", so do NOT write
  a Case/line/resolves line yourself. HARD LIMIT: setup <= 175 characters.

oracle_open — Oracle's opening, replying to the setup: its stance in words plus
  ONE main argument (its single strongest factor), 30-40 words. If Oracle LOST
  the last resolved day (see track_records / last_result), OPEN with a one-phrase
  admission, e.g. "Called it wrong yesterday, but —". HARD LIMIT <= 280 characters.

guardian_attack — Guardian's reply, which MUST ATTACK ORACLE'S SPECIFIC ARGUMENT
  from oracle_open — attack the argument, not the topic. Find its weak point (thin
  data, how long the signal has ALREADY run without paying off, a base rate that
  cuts the other way, a cross-asset read) and hit THAT, then land your own
  counter-thesis. Not a parallel method dump. 30-40 words. HARD LIMIT <= 280 chars.

oracle_jab — Oracle's last word: a short counter-jab, 15-20 words. Ends on its
  own terms — NO reconciliation, no "we'll see". HARD LIMIT <= 160 characters.

card_caption — ONE line for the bet card: a punchy hook, no numbers. <= 120 chars.

predictions — two mutually exclusive calls at ONE watershed level: the price
  above which Oracle is right and below which Guardian is right, at the nearest
  structural level (yesterday's high/low from data_payload, a range boundary, or
  a round number). It MUST NOT equal the current price (±15% max). Each carries a
  "driver": one plain reason (≤55 chars, no jargon, no $) shown on the card.

FORBIDDEN in oracle_open / guardian_attack / oracle_jab: ANY $ amount or %
(every number lives on the card — the arguments are words), the words
"guaranteed"/"will"/any promise, parallel monologues, polite hedges, and service
words (quote, past_calls, verbatim, track_record). The setup may name the current
price; the reply messages may not carry figures.
