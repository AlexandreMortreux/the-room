# THE ROOM

Автономный ежедневный Telegram-дайджест по Bitcoin. Два AI-аналитика с
противоположными методологиями разбирают один рыночный сигнал дня, фиксируют
взаимоисключающие проверяемые прогнозы и ведут публичный счёт точности.
Всё работает на GitHub Actions по крону — без серверов.

## Как это устроено

- **🔮 Oracle** — макро-аналитик: режимы ликвидности, контрарные сигналы,
  экстремумы страха как возможности.
- **🛡 Guardian** — риск-аналитик: инерция потоков, базовые частоты,
  требование подтверждения.

Каждый день в 09:00 (UTC+8) прогон:

1. Резолвит вчерашние прогнозы по дневному закрытию BTCUSDT на Binance
   и постит развязку со счётом.
2. Собирает новости (MarketTwits) и рыночные данные (цены, funding,
   открытый интерес, Fear & Greed) — все источники бесплатные, без ключей.
3. Классификатор (Claude Haiku) выбирает сигнал дня; если сигнала нет —
   fallback-выпуск (миф, пересуд старого спора или разбор позиционирования).
4. Дебат-движок (Claude Sonnet) генерирует пост: сигнал, 4–6 реплик спора,
   вывод-режим и два взаимоисключающих прогноза по одному уровню
   (горизонт 24 часа, уверенность 0.55–0.80).
5. Публикует пост и опрос «Кто прав завтра?» в Telegram-канал, дописывает
   прогнозы в `ledger.csv` и коммитит его в репозиторий.
6. По воскресеньям (UTC+8) дополнительно постит scorecard недели.

## Как проверить леджер

`ledger.csv` — единственный источник правды, каждая строка — один прогноз:

| Поле | Значение |
|---|---|
| `direction`, `level` | «выше/ниже уровня» на момент экспирации |
| `expires_utc` | момент истечения горизонта (24 ч от публикации) |
| `price_at_expiry` | ближайшее **дневное закрытие** BTCUSDT на Binance после `expires_utc` |
| `result` | `pending` / `win` / `loss` |
| `brier` | `(confidence − outcome)²`, где outcome = 1 при win, 0 при loss |

Проверка любой строки вручную: возьмите дневную свечу BTCUSDT
(`GET https://api.binance.com/api/v3/klines?symbol=BTCUSDT&interval=1d`),
найдите первое закрытие после `expires_utc` и сравните с `level`.
История изменений леджера — в git-логе: каждая запись и каждый резолв
зафиксированы отдельным коммитом бота, задним числом ничего не меняется.

## Запуск

Секреты репозитория (Settings → Secrets and variables → Actions):

- `ANTHROPIC_API_KEY` — ключ Anthropic API;
- `TG_BOT_TOKEN` — токен Telegram-бота;
- `TG_CHANNEL_ID` — id канала (например `@the_room_btc` или `-100…`).

Расписание — `.github/workflows/daily.yml` (cron `0 1 * * *` = 09:00 UTC+8),
ручной запуск — через `workflow_dispatch`.

### Локальная отладка (DRY_RUN)

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
DRY_RUN=1 python main.py
```

В этом режиме сообщения печатаются в stdout вместо отправки в Telegram,
а леджер пишется в `ledger_dry.csv` (в git не попадает).

## Критерии успеха

Kill-test на 30-й день зафиксирован заранее в [`thresholds.md`](thresholds.md):
спрос (подписчики, явка в опросах, удержание просмотров) решает go/no-go,
точность и Brier из `ledger.csv` только объясняют результат.

## Дисклеймер

Не является инвестиционной рекомендацией. Прогнозы — публичный эксперимент
по калибровке AI-аналитиков; весь трек-рекорд открыт в `ledger.csv`.
